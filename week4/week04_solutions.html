2. High Performance Haversine

In this exercise, we will expand on the example from section 2.2 in Fast Python about profiling code for the Haversine distance. Consider the following Python program:

import sys
import numpy as np

def distance_matrix(p1, p2):
    p1, p2 = np.radians(p1), np.radians(p2)

    D = np.empty((len(p1), len(p2)))
    for i in range(len(p1)):
        for j in range(len(p2)):
            dsin2 = np.sin(0.5 * (p1[i] - p2[j])) ** 2
            cosprod = np.cos(p1[i, 0]) * np.cos(p2[j, 0])
            a = dsin2[0] + cosprod * dsin2[1]
            D[i, j] = 2 * np.arctan2(np.sqrt(a), np.sqrt(1 - a))

    D *= 6371  # Earth radius in km
    return D


def load_points(fname):
    data = np.loadtxt(fname, delimiter=',', skiprows=1, usecols=(1, 2))
    return data


def distance_stats(D):
    # Extract upper triangular part to avoid duplicate entries
    assert D.shape[0] == D.shape[1], 'D must be square'
    idx = np.triu_indices(D.shape[0], k=1)
    distances = D[idx]
    return {
        'mean': float(distances.mean()),
        'std': float(distances.std()),
        'max': float(distances.max()),
        'min': float(distances.min()),
    }


fname = sys.argv[1]
points = load_points(fname)
D = distance_matrix(points, points)
stats = distance_stats(D)
print(stats)

This program loads a set of latitude and longitude coordinates, computes a distance matrix using the Haversine distance, and finally prints some summary statistics.

  6. Optimize distance_matrix to improve its performance. There are at least two optimizations you can make:

    1. Moving a repeated calculation out of the loop to pre-compute once.
    2. The fact that \arctantwo(a,1−a)=arcsin(a). Re-run the profiler. Did it make the code faster? Did it change where time is spent in distance_matrix?

    Answer: Instead of computing the cosine of individual elements every time in line 28, we can pre-compute it for all elements before the loop. This has two advantages:

    1. We do not re-compute the cosine of the p2 elements every iteration, which is wasted work.
    2. We take advantage of NumPy array operations by only calling np.cos once instead of incurring the overhead of function calls every loop iteration.

    Next, instead of computing the final result with np.arctan2(np.sqrt(a), np.sqrt(1 - a)), we can use the identity above and use np.arcsin(np.sqrt(a)). This means we only compute one square root, which saves time.

    For me, with an Intel(R) Xeon(R) Gold 6226 CPU @ 2.70GHz and with all locations as input, these two optimizations reduced the runtime of distance_matrix from 12.6 seconds to 8.5 which is about 25% faster. Not mind blowing, but certainly worth the work.
